What This Script Does

  • Processes batch input (one item per line) and supports numeric PMID, PMCID, PubMed URL, and PMC URL formats.
  • Normalizes each line to PMID/PMCID, then resolves to a PMC record when available.
  • For PMC articles, fetches full-text XML and scans text with ENA accession regex patterns.
  • Shows per-item output plus batch summary counts (parsed, invalid, PMC coverage, matches, and request errors).
  • Applies client-side request throttling (~400 ms minimum between Entrez calls) to stay under no-key limits.

Export and metadata behavior

CSV export is one row per input article. Accessions are deduplicated and stored in one cell as a semicolon-separated list with an accession count.

Metadata mode adds: title, journal, publication year, DOI, and authors. Export uses the metadata mode from the batch run itself (not the current checkbox state), so schema/content stay consistent with that run.

ENA accession format reference: ENA accession documentation

Case 1: Not a PMC article
Status: This is not a PMCID article.
Meaning: The PMID does not have a `pubmed_pmc` link, so no full-text fetch is attempted.
Case 2: PMC article but no accession codes
Status: PMCID found (...). No ENA accession matches found.
Meaning: Full text is available, but none of the current ENA regex patterns matched text in this paper.
Case 3: PMC article with accession codes
Status: PMCID found (...). N accession(s) matched.
Meaning: One or more ENA-style accession strings were detected and listed in the output panel.
